1.2 - Was ist LEdit und was kann es?

LEdit ist ein universeller Editor für Lufia 2 - Rise of the Sinistrals (in Deutschland schlicht als "Lufia" bekannt). Es soll Leuten, die keine Ahnung vom ROM-Hacking haben, die Möglichkeit geben, eigene Inhalte für dieses Spiel zu erstellen und in einen ROM zu patchen, so dass es weiterhin mit Emulatoren lauffähig ist.

Hierbei ist zu beachten, dass viele Dinge nur bearbeitet, jedoch nicht neue hinzugefügt werden können.
In einigen Bereichen wird das zwar in Zukunft möglich sein, jedoch ist es bei vielen Dingen aufgrund technischer Begrenzungen des Super Nintendo leider nicht möglich.
Um LEdit überhaupt starten zu können, wird das Microsoft .NET Framework 2.0 oder höher benötigt.

Es wird nur der deutsche ROM unterstützt, was sich in Zukunft jedoch noch ändern kann.
Folgende Funktionen sind bereits verfügbar:
Monster bearbeiten
Sämtliche Werte der Monster (Level, Stärke, Gold, Erfahrung usw.) können verändert werden.
Für die ganz Harten ist es auch möglich, die Kampf-Skripts zu bearbeiten, so dass sogar das Verhalten im Kampf modifizierbar ist.

MonsterSprites bearbeiten
Als "MonsterSprites" bezeichne ich die Grafiken der Monster im Kampf.
Der MonsterSprite-Editor kann 16-Farben-Bitmaps (bmp) exportieren und importieren. Kurzum bedeutet dies, dass man sich neue Monster in Paint oder dergleichen malen kann. Darüberhinaus können Farbpaletten verändert und sogar neue hinzugefügt werden.

MapSprites bearbeiten
"MapSprites" habe ich die Grafiken von NPCs in Dungeons getauft, das sind beispielsweise Monster oder Maxim selbst.
Auch hier können 16-Farben-Bitmaps exportiert und wieder importiert werden, so dass das Arbeiten mit Paint oder dergleichen möglich ist.
Die erweiterten Daten können lediglich betrachtet werden, weil deren Funktionen teilweise noch ungeklärt sind.

Maps bearbeiten
Was wäre ein Editor wie dieser ohne einen Karten-Editor?
Sämtliche "Dungeon-Karten" (Dungeons, Türme, Städte etc.) können bearbeitet werden:
  • Auf beiden Tilemap-Layern einer Karte kann mit einem gegebenen Satz an Blöcken frei gezeichnet werden.
  • Die Kopier- und Einfüge-Funktion erleichtert das Erstellen neuer Orte ungemein.
  • Die Rückgängig-Funktion kann Fehler schnell rückgängig machen.
  • Auch Kollisionen ("Wände") und weitere erforderlichen Dinge können bearbeitet werden.
  • Karten können vergrößert werden, um sie zu erweitern oder einfach mehr Platz zu haben.
  • Einfache Türen können definiert werden.
  • So genannte "Warp-Türen", die den Spieler auf eine andere Karte teleportieren, können gesetzt werden.
  • Die Startpunkte und Bewegungsbereiche von Charakteren bzw. Monstern (in Dungeons) können begrenzt bearbeitet werden.

Die Weltkarte kann zur Zeit nur betrachtet werden (nützlich für Warp-Türen und einfach nur geil), die Unterwasserkarte ist noch gar nicht verfügbar.

Dateien extrahieren und importieren
Dies ist eher was für ROM-Hacker (also speziell für mich selbst).
Die komprimierten Dateien können aus dem ROM zur Bearbeitung extrahiert und später wieder importiert werden.

Dies ist Grundlage aller Editoren, jedoch machen diese das automatisch und für den Benutzer unsichtbar.

Items
Sämtliche Daten aller Items (Kraftwasser, Waffen, Rüstungen, etc.) können bisher leider nur betrachtet werden.

1.3 - LPatch

Da das Spiel nicht dafür ausgelegt ist, bearbeitet zu werden, gibt es einige Restriktionen, die vorher beseitigt werden müssen.
Dies habe ich mit dem LPatch realisiert. Er verändert den ROM in folgender Weise:
  • Der ROM (LoROM) wird zu einem ExLoROM erweitert, er ist dann 8MB groß.
  • Damit Lufia unkomprimierte Dateien nutzen kann, wird ein Code-Hack installiert.
  • Weitere Code-Hacks werden installiert, damit verschobene Daten korrekt geladen werden (z.B. Monster).
  • Ist der (ohnehin unnütze) ROM-Header vorhanden, wird dieser entfernt.
Die Funktionalität wird hierdurch in keinster Weise beeinträchtigt.
LEdit installiert den Patch nur, wenn dies bestätigt wird und erstellt ein Backup des Originals. Letzteres sollte zur absoluten Sicherheit jedoch zusätzlich vorher manuell erfolgen.

1.4 - Der "files"-Ordner

Nach Installation des LPatch werden auch bereits einige Daten extrahiert.

Binäre Daten werden in die Verzeichnisstruktur unter files abgelegt.
Wenn man beispielsweise ein Monster im Monster-Editor auswählt, wird, so fern dies noch nicht geschehen ist, die Monster-Grafik aus dem ROM extrahiert und in files\monstersprites abgelegt. Wird hierfür nun eine Bitmap importiert oder eine Palette hinzugefügt und Speichern gewählt, wird in die Datei gespeichert, noch nicht in den ROM.

Dies hat zwei Gründe: Geschwindigkeit und vor allem Sicherheit.
Wenn Daten im ROM erst einmal überschrieben sind, lassen sie sich nicht wiederherstellen. Daher sollte man Daten nur in den ROM einfügen, wenn man sich da ganz sicher ist.

Dieses Konzept wird von allen Editoren verfolgt. Erst durch Datei → In ROM einfügen werden Daten auch tatsächlich im Spiel verfügbar gemacht.

Wird der Ordner files gelöscht, sind alle diese temporären Daten verloren.
Dies kann sinnvoll sein, wenn man Mist gebaut hat und die Daten aus dem ROM wiederherstellen möchte, denn alles was nicht in files gefunden wird, wird wieder aus dem ROM extrahiert.

1.5 - Die XML-Datei

Es besteht die ausdrückliche Empfehlung, diese Datei NICHT zu bearbeiten.

Sie enthält Meta-Informationen, die nichts mit dem Spiel direkt zu tun haben, sondern für LEdit wichtig sind.

In der Vorab-Version wird nur die Datei Lufia 2.smc geladen, die sich im gleichen Ordner wie LEdit befinden muss. Wird diese nicht gefunden, geschieht absolut gar nichts

Wenn der LPatch noch nicht installiert ist, wird die Installation angeboten.

Sie folgen dem "mir nichts dir nichts"-Prinzip, d.h. wenn man sie in Ruhe lässt, lassen sie einen auch in Ruhe. Das Bearbeiten wird ausdrücklich nicht empfohlen.

2.2 - Monster-Editor

Mit dem Monster-Editor können Monsterdaten bearbeitet werden. Er sieht folgendermaßen aus:


Links, im Feld Monster, sind alle Monster des Spiels, sortiert nach interner ID-Nummer, aufgelistet.
Um eins zu bearbeiten, wird dieses aus der Liste ausgewählt.
In Feld Ansicht unter der Liste findet sich eine Vorschau der Grafik im Kampf (MonsterSprite) sowie die verwendete Palette. Ein Doppelklick auf die Vorschau öffnet den → MonsterSprite-Editor für diese Grafik.

Das Feld Eigenschaften stellt den eigentlichen Editor dar.
Alle Daten des Monsters werden hier zur Bearbeitung bereitgestellt, die Beschreibungen sollten dabei klar sein, hier einige besondere:
???
Den Nutzen dieser Eigenschaft habe ich noch nicht herausgefunden.
Bei den meisten Monstern ist der Wert 50, er weicht jedoch auch teilweise ab (z.B. Panzergoblin mit 54).

Sollte jemand herausfinden, für was dieser Wert ist, bitte ich diesen, mich zu kontaktieren.
Sprite

Aus irgendeinem Grunde bin ich noch nicht dazu gekommen, diesen Wert veränderbar zu machen.
Palette
Dies ist die Palette des jeweiligen MonsterSprites, die für dieses Monster verwendet werden soll.
Wenn man sich die drei Blobs ansieht, sollte der Nutzen klar werden.

HINWEIS: Fügt man im MonsterSprite-Editor eine Palette hinzu, weiß der Monster-Editor erst dann davon, wenn man das Sprite neu lädt. Monster-Editor zu und wieder auf ist die einfachste Methode.

Dies ist das Item, welches man erhalten kann, wenn man dieses Monster im Kampf schlägt.
Hierzu gehört auch die Wahrscheinlichkeit, mit der man dieses Item erhält.

Das Attacke-Skript wird vom Spiel ausgeführt, wenn das Monster mit angreifen dran ist.
Das Defensiv-Skript wird ausgeführt, wenn das Monster Schaden erleidet.

Ansehen... öffnet den → Skript-Editor.

Die Menü-Einträge haben folgende Funktionen:
Monster → Speichern (STRG+S)
Speichert die Daten aller Monster in die Datei files\monsters.l2.
Es ist empfehlenswert, nach jeder Bearbeitung eines Monsters zu speichern.

Wird der Editor geschlossen, wenn ein Monster bearbeitet wurde, wird gefragt, ob gespeichert werden soll.
Monster → Alle in ROM einfügen... (STRG+I)
Fügt die Daten aus files\monsters.l2 in den ROM ein, so dass sie im Spiel vorhanden sind.

2.3 - MonsterSprite-Editor

Der MonsterSprite-Editor ist ein Tool zum Bearbeiten der Kampf-Grafiken von Monstern:


Links ist die Liste aller MonsterSprites des Spiels, sortiert nach interner ID-Nummer.
Es werden nicht außerordentlich viele, dafür jedoch sehr mächtige Funktionen geboten:
Tilemap → BMP Exportieren
Speichert die Grafik in eine 16-Farben-Bitmap (bmp), welche mit gängigen Programmen wie Paint, Photoshop oder ähnlichen geöffnet und bearbeitet werden kann.
Tilemap → BMP Importieren
Lädt eine 16-Farben-Bitmap (bmp) und konvertiert sie in eine Super Nintendo-Tilemap, die Lufia verwenden kann.

Standardmäßig exportiert man eine Bitmap, bearbeitet sie und importiert sie wieder.
Importiert man eine eigene Bitmap, ist die Größe zu beachten:
Sie muss mindestens 32x32 und darf höchstens 128x128 groß sein, dabei müssen sowohl Höhe als auch Breite durch 16 teilbar sein.

Außerdem ist zu beachten, dass der Super Nintendo nur begrenzt Video-Speicher aufweist.
Kommt man auf die Idee, einem Blob die Grafik von Gades zu geben und trifft im Spiel auf 4 Blobs, wird man dafür bitter bestraft...

Nur die Tilemap wird importiert, nicht die Palette. Diese muss, wenn nötig, gesondert geladen werden (siehe unten).
Palette → Hinzufügen
Fügt dem MonsterSprite eine neue Palette als Kopie der momentan gewählten Palette hinzu.
Palette → Entfernen
Entfernt die momentan gewählte Palette, sofern sie nicht die einzige ist.
Palette → Aus BMP Importieren
Lädt eine 16-Farben-Bitmap und konvertiert die 16 Farben in eine Palette. Die momentan gewählte Palette wird dabei überschrieben.
Die erste Farbe einer Palette wird im Spiel nicht dargestellt.
Datei → Speichern (STRG+S)
Speichert das ausgewählte MonsterSprite in die entsprechende Datei unter files\monstersprites.
Datei → In ROM einfügen... (STRG+I)
Fügt das gewählte MonsterSprite in den ROM ein und installiert es damit ins Spiel.
Datei → Wiederherstellen (STRG+I)
Nicht in dieser Version verfügbar.

2.5 - Map-Editor

Mit dem Map-Editor werden die Karten des Spiels bearbeitet.
Was Karten betrifft, gibt es aus technischer Sicht noch sehr viel Forschungsbedarf, weshalb noch nicht all zu viel möglich ist. Vieles scheint noch sehr provisorisch, kann aber schon vieles ausrichten.

Der Editor sieht wie folgt aus:


Zunächst muss eine Karte geladen werden. Hierzu wählt man die gewünschte Karte aus der Liste aus und wählt Map laden.
Alternativ kann man die Weltkarte laden und betrachten, sie kann jedoch noch nicht bearbeitet werden.

Im Feld Map wird die Karte gezeichnet. Mithilfe der Scrollbalken kann man über die Karte scrollen.
Die Statusleiste zeigt einige Informationen, von links nach rechts:
  • Name der Karte
  • Größe der Karte (Breite x Höhe) in Tiles
  • Blockset-Nummer
  • Tileset-Nummer
  • Maus-Koordinaten (x, y) in Tiles
  • ID des Tiles unter dem Mauszeiger auf Layer 1
  • ID des Tiles unter dem Mauszeiger auf Layer 2
Lediglich die Maus-Koordinaten sind wohl für den Nutzer von Interesse.

Der Editor arbeitet modusbasiert, das heißt abhängig davon, in welchem Modus man sich befindet, werden Dinge in der Ansicht gezeichnet und hat die Maus unterschiedliche Funktionen.
Der Modus kann im Menü Bearbeiten oder in der Toolbar gewählt werden.
Auswählen
In diesem Modus können Tiles ausgewählt werden.
Mit der linken Maustaste wird ein Tile ausgewählt, auch mehrere können ausgewählt werden.
Mit der mittleren Maustaste wird im Feld Blöcke das Tile an der Mauspoistion auf dem gewählten Layer selektiert (vergleichbar mit der Pipette in Paint).
Die rechte Maustaste öffnet das Kontextmenü, welches einige Operationen für die gewählten Tiles bietet.

So lassen sich beispielsweise die gewählten Tiles kopieren und woanders wieder einfügen.
Charakter-Bereiche, Türen und Warp-Türen werden später erläutert.

Die Funtkion Raum erstellen funktioniert derzeit nur in Höhlen (z.B. Höhle der Geheimnisse) und Tempeln (z.B. Schwerttempel). Wurde eine Fläche gewählt, wird diese mit dem Standard-Boden-Tile gefüllt und Wände drum herum gezeichnet.
Ist der Raum fertig, kann man auf ein Wand-Tile klicken und Wände verbinden wählen, um die Ecken etc. zu einer kontinuierlichen Wand zu verbinden.

Ein Beispiel: Schritt 1 · Schritt 2 · Schritt 3
Zeichnen
In diesem Modus können Tiles gezeichnet werden.
Mit dem Tile, das im Feld Blöcke ausgewählt ist, wird auf dem gewählten Layer (1 oder 2) gezeichnet.

Die linke Maustaste hat keine Funktion. Dies kann sich ändern, jedoch klickt man mal gern aus Versehen irgendwohin und daher wurden der Taste jegliche Funktionen entzogen.
Die mittlere Taste fungiert wie im Auswahl-Modus als Pipette,
mit der rechten Maustaste wird gezeichnet.
Fläche füllen
Funktioniert noch nicht.
Horizontale / Vertikale Kollisionen bearbeiten
Wird dieser Modus gewählt, wird man rote Linien in der Ansicht sehen. Diese zeichnen Kollisionen, das heißt Linien, über die der Spieler sich nicht bewegen kann, es sind so zu sagen Wände.
Jedes Tile besitzt eine Kollision links und oben, vertikal und horizontal.

In diesem Modi kann man mit der linken Maustaste Kollisionen setzen und mit der rechten entfernen.

Beispiel: Schritt 1 · Schritt 2
Raumgrenzen bearbeiten
Diese Funktion ist nur für Innenkarten sinnvoll.
Ist sie aktiviert, können Raumgrenzen gesetzt und entfernt werden, sie werden in Form von blauen X-Zeichen dargestellt.
Raumgrenzen werden von Lufia benötigt, um zu "wissen", wo es aufhören soll, den Raum zu zeichnen.
Ein gutes Beispiel ist Karte 0x04 - Taia.
Raumübergänge bearbeiten
Die tatsächliche Funktion von Raumübergängen ist mir noch nicht bekannt, jedoch findet man diese überall an Türen an Stelle von Raumgrenzen.
Sie sollten nur bei der Erstellung von Türen gesetzt werden und sind ebenfalls nur für Innenkarten sinnvoll.
Sie werden als grüne X-Symbole dargestellt.
Hindernisse bearbeiten
Hindernisse sind diejenigen Tiles, an denen beispielsweise Pfeile abprallen oder das Schwert ein entsprechendes Geräusch macht. Eine konkrete Funktion ist mir noch nicht bekannt (das oben genannte hätte man ja auch mit Kollisionen realisieren können).
Sie machen nur in Dungeons Sinn und werden durch ein gelbes Plus-Symbol dargestellt.
Mehrfach erwähnt wurden bereits die Layer.
Eine Karte besteht aus zwei Tilemap-Layern. Layer 1 wird zuerst gezeichnet, danach die Sprites (z.B. Maxim) und zuletzt Layer 2,
Layer 1 ist damit also der Hintergrund und Layer 2 der Vordergrund.
Welcher Layer bearbeitet wird, kann mit den entsprechenden Buttons gewählt werden.
Kollisionen etc. sind vom Layer komplett unabhängig.

Die Gitternetzlinien erklären sich von allein. Über die Karte wird dann ein Gitternetz gezeichnet, was bei präzisen Zeichnungen helfen kann.

In der Toolbar gibt es noch weitere Ansichtsoptionen:
Türen
Türen sind spezielle Bereiche, in denen
  • die Tiles geändert werden,
  • die Kollision aufgehoben wird,
wenn man sie berührt.

Eine Tür besteht in Lufia aus drei Bereichen:
  1. Der Bereich der Tür-Tiles, die sich bei Öffnung ändern (rot im Editor).
  2. Unbekannte Funktion...
  3. Unbekannte Funktion...

Außerdem besitzen sie zwei weitere noch unbekannte Eigenschaften.
Warp-Türen
Warp-Türen sind Bereiche, die, wenn man sie betritt, den Spieler an einen anderen Ort teleportieren.
Ein Beispiel ist der Rand einer Stadt: läuft man darüber hinaus, kommt man auf die Weltkarte. Treppen usw. funktionieren auf diese Weise.
Charakter-Bereiche
Diese beschreiben Startpunkt und Bewegungsbereich eines NPCs dar, also beispielsweise eines Monstes oder der Berwohner in der Stadt.
Es ist leider nicht geklärt, wie festgelegt werden kann, welche Art von NPC an diese Stelle gesetzt wird.
Trigger
Trigger sind Bereiche, die, wenn man sie betritt, ein Story-Ereignis auslösen.
Ein Beispiel auf Karte 0x03 - Taia: Betritt man am Anfang des Spiels Trigger-Bereich 0x02, kommt der alte Herr und bestellt den Spieler in die Höhle der Geheimnisse.

Trigger können noch nicht bearbeitet werden, weil sie weitestgehend unerforscht sind.
Mittels Map → Größe ändern... lässt sich zuletzt die Größe der Karte in Tiles verändern.
Wird sie vergrößert, wird der neue Bereich mit Null-Tiles belegt, wird sie verkleinert, wird der überflüssige Bereich abgeschnitten (also Vorsicht!).

Die Funktionen Datei → Speichern und Datei → In ROM einfügen... funktionieren wie in den anderen Editoren.
Karten befinden sich in files\maps.

2.8 - Skript-Editor

Der Skript-Editor ermöglicht es, Kampf-Skripte von Monstern oder Items zu bearbeiten.
Diese liegen in einem Lufia 2-eigenen Bytecode vor. Der Skript-Editor übersetzt diesen Code in eine von mir vorübergehend entwickelte Assembler-ähnliche Sprache, damit dieser lesbarer wird.
Bearbeiten kann man den Code jedoch vorerst nur in Bytes.


Folgende Menüfunktionen sind verfügbar:
Skript → Code kompilieren
Versucht, den Bytecode aus dem Textfeld Code zu interpretieren und gibt den neuen Code "übersetzt" im Feld Skript aus.
Skript → Speichern
Speichert den kompilierten Code (z.B. in die Daten des Monsters, das gerade bearbeitet wird.
